## 1. 개발 서버에 CMS.API 배포

> [!info] 배경
> - 클라이언트 앱과 서버 간 통신 테스트를 위해 개발 서버에 API 배포 필요
> - 서버 정보: 210.219.229.46 (Windows Server 2016)
> - 기존 서비스: 8080(wbs), 2121(ftp) 사용 중

> [!warning] 문제
> - ffmpeg.exe 파일 누락으로 Visual Studio 게시 실패
> - 웹사이트를 FTP 사이트로 잘못 생성 (HTTP 바인딩이 아닌 FTP)
> - 방화벽 규칙을 아웃바운드에 생성 (인바운드가 아닌)
> - 외부에서 접속 시 ERR_CONNECTION_TIMED_OUT 발생

> [!success] 해결
> - FTP 사이트 삭제 후 "Add Website"로 HTTP 웹사이트 재생성
> - 아웃바운드 규칙 삭제, 인바운드 규칙으로 8081 포트 추가
> - 응용 프로그램 풀을 .NET v4.5로 변경
> - 외부 접속 성공: http://210.219.229.46:8081

> [!tip] 배운 점
> - IIS에서 "Add Website" vs "Add FTP Site" 구분 필요
> - 방화벽 인바운드(외부→서버) vs 아웃바운드(서버→외부) 차이
> - 서버가 공유기 뒤에 있으면 포트 포워딩 필요 (이 경우는 직접 연결)

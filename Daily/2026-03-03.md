## 2. SamsungWelstory (백엔드 서버)

### 기술 스택

| 구분 | 기술 |
|------|------|
| Framework | ASP.NET MVC 5.2.7, Web API 5.2.4 |
| Language | C# (.NET Framework 4.5.2) |
| Database | Oracle Database (ODP.NET) |
| Authentication | OWIN, ASP.NET Identity |
| Frontend | jQuery 3.3.1, Bootstrap, Font Awesome |

### 프로젝트 구성

| 프로젝트 | 용도 |
|----------|------|
| **CMS.API** | 메인 웹 서버 (콘텐츠 관리 시스템) |
| **ConvertMP4** | 동영상 MP4 변환 유틸리티 |
| **ScreenShotUrl** | URL 스크린샷 캡처 유틸리티 |
| **ScreenShotMovie** | 동영상 썸네일 추출 유틸리티 |

### CMS.API 주요 기능

| 컨트롤러 | 기능 |
|----------|------|
| AccountController | 로그인/인증 |
| DisplayController | DID 장비 관리 |
| ContentsController | 콘텐츠(이미지/동영상) 관리 |
| LayoutController | 화면 레이아웃 설정 |
| TemplateController | 콘텐츠 템플릿 관리 |
| ScheduleController | 시간대별 스케줄 관리 |
| SensorController | 센서 데이터 수집 |
| UserController | 사용자/권한 관리 |

## 3. WelStoryDIDClient (Windows 클라이언트)

### 기술 스택

| 구분 | 기술 |
|------|------|
| Framework | .NET Framework 4.5 |
| UI | Windows Forms |
| Browser Engine | CefSharp (Chromium Embedded) |
| JSON | Newtonsoft.Json 11.0.2 |

### 프로젝트 구성

| 프로젝트 | 용도 |
|----------|------|
| **WelStoryDIDClient** | 메인 클라이언트 (DID 장비에 설치) |
| **DIDClientConfig** | 클라이언트 설정 유틸리티 |
| **SetConfig** | 설정 공유 라이브러리 |
| **DIDClientSetup** | MSI 설치 프로젝트 |

### 클라이언트 주요 기능

- CMS 서버에서 스케줄 정보 주기적 조회
- Chromium 엔진으로 웹 콘텐츠 렌더링
- 모니터 전원 On/Off 제어
- 자동 재부팅/종료 스케줄
- 카드리더 연동 (시리얼 포트)
- 멀티 스크린 지원

### 클라이언트 설정 항목 (Config.xml)

```xml
<Config>
  <WebServiceURL>http://[CMS서버주소]</WebServiceURL>
  <StoreID>매장코드</StoreID>
  <ShutdownUse>true</ShutdownUse>
  <ShutdownH>23</ShutdownH>
  <ShutdownM>00</ShutdownM>
  <RebootUse>true</RebootUse>
  <RebootH>06</RebootH>
  <RebootM>00</RebootM>
  <CardReader>false</CardReader>
  <ComPort>COM1</ComPort>
</Config>
```

## 4. 시스템 아키텍처

```
┌─────────────────────────────────────────────────────┐
│                    CMS.API (서버)                    │
│  ASP.NET MVC + Web API + Oracle DB                  │
│  - 콘텐츠/스케줄/레이아웃/템플릿 관리                │
│  - 사용자 인증 및 권한 관리                          │
│  - 디스플레이 장비 모니터링                          │
└──────────────────────┬──────────────────────────────┘
                       │ REST API (JSON)
          ┌────────────┴────────────┐
          ▼                         ▼
┌─────────────────────┐   ┌─────────────────────┐
│  Windows Client     │   │  Android Client     │
│  (WelStoryDIDClient)│   │  (welstoryDID.apk)  │
│  - CefSharp 렌더링  │   │                     │
│  - 디바이스 제어    │   │                     │
└─────────────────────┘   └─────────────────────┘
```

### 백엔드 서버 vs 클라이언트 용도 차이

| 구분 | 백엔드 서버 (SamsungWelstory) | 클라이언트 (WelStoryDIDClient) |
|------|------------------------------|-------------------------------|
| **설치 위치** | 회사 서버 (1대) | DID 장비마다 설치 (N대) |
| **사용자** | 관리자 | 무인 (자동 실행) |
| **접속 방법** | 웹브라우저 | Windows 프로그램 |
| **주요 역할** | 콘텐츠/스케줄 관리 | 콘텐츠 표시 |
| **데이터** | DB에 저장 | 서버에서 받아옴 |

> [!info] 쉽게 말해
> - **서버**: 관리자가 "뭘 보여줄지" 설정하는 곳
> - **클라이언트**: 실제 화면에 "보여주는" 프로그램

### 실제 사용 흐름

```
[관리자 - 백엔드 서버]
1. 웹브라우저로 CMS 접속
2. 새 이미지/동영상 업로드
3. "오전 11시~오후 1시에 점심메뉴 표시" 스케줄 설정
4. 저장
        │
        ▼
[DID 장비 - 클라이언트]
1. 서버에서 스케줄 정보 주기적으로 확인
2. 11시가 되면 자동으로 점심메뉴 화면 표시
3. 1시가 되면 다음 콘텐츠로 전환
```

## 7. Visual Studio에서 솔루션 열기

### 솔루션 파일 위치

| 프로젝트 | 경로 |
|----------|------|
| 백엔드 서버 | `C:\wonjun\DID\DID\03.DID_solution\SamsungWelstory\SamsungWelstory.sln` |
| 클라이언트 | `C:\wonjun\DID\DID\03.DID_solution\WelStoryDIDClient\WelStoryDIDClient\WelStoryDIDClient.sln` |

### 열기 방법

1. **파일 탐색기에서**: `.sln` 파일 더블클릭
2. **Visual Studio에서**: `Ctrl + Shift + O` → 파일 선택
3. **시작 화면에서**: "프로젝트 또는 솔루션 열기" 클릭

> [!tip] 먼저 백엔드 서버(`SamsungWelstory.sln`)를 열어보는 것을 권장

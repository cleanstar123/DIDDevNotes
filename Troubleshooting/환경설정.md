## 1. Visual Studio 2022에서 .NET Framework 4.5.2 타겟팅 팩을 찾을 수 없음

### 문제 상황

> [!warning] 증상
> - Visual Studio Installer의 "개별 구성 요소"에서 검색해도 나오지 않음
> - `.NET Framework 4.5.2 타겟팅 팩`
> - `.NET Framework 4.6.1 타겟팅 팩`
> - `.NET Framework 4.5.2 SDK`

### 원인

> [!info] 원인
> Visual Studio 2022는 .NET Framework 4.6.2 이상만 기본 지원
> .NET Framework 4.5.x는 레거시로 분류되어 별도 설치 필요

### 해결 방법

#### 방법 1: Developer Pack 별도 설치 (권장)

Microsoft 공식 사이트에서 직접 다운로드:

| 버전 | 다운로드 URL |
|------|--------------|
| .NET Framework 4.5.2 Developer Pack | https://dotnet.microsoft.com/download/dotnet-framework/net452 |
| .NET Framework 4.6.1 Developer Pack | https://dotnet.microsoft.com/download/dotnet-framework/net461 |

**설치 순서:**
1. Visual Studio 및 Installer 종료
2. .NET Framework 4.5.2 Developer Pack 다운로드
3. 설치 파일 실행 (NDP452-KB2901954-Web.exe 등)
4. .NET Framework 4.6.1 Developer Pack도 동일하게 설치
5. PC 재시작
6. Visual Studio에서 솔루션 열기

#### 방법 2: 프로젝트 타겟 버전 업그레이드

프로젝트를 최신 .NET Framework로 업그레이드:

| 현재 | 업그레이드 대상 | 호환성 |
|------|----------------|--------|
| 4.5.2 | 4.7.2 또는 4.8 | 대부분 호환 |
| 4.6.1 | 4.7.2 또는 4.8 | 대부분 호환 |

**업그레이드 방법:**
1. 솔루션 열기 (경고 무시하고 열기)
2. 각 프로젝트 우클릭 → 속성
3. 대상 프레임워크 → `.NET Framework 4.7.2` 또는 `4.8` 선택
4. NuGet 패키지 복원
5. 빌드 테스트

> [!tip] 권장사항
> 신규 환경 구축이므로 방법 2(업그레이드)를 권장
> .NET Framework 4.8은 장기 지원 버전이며 보안 패치 지속 제공

### Visual Studio 2022 기본 지원 버전

| 버전 | 지원 여부 |
|------|----------|
| .NET Framework 4.5.x | 별도 설치 필요 |
| .NET Framework 4.6.x | 별도 설치 필요 |
| .NET Framework 4.7.x | 기본 지원 |
| .NET Framework 4.8 | 기본 지원 |

## 2. Visual Studio 필수 워크로드

### DID 프로젝트에 필요한 워크로드

Visual Studio Installer에서 선택:

```
☑ ASP.NET 및 웹 개발
    └─ CMS.API 백엔드 서버용

☑ .NET 데스크톱 개발
    └─ WelStoryDIDClient Windows Forms 클라이언트용
```

### 개별 구성 요소 (확인 필요)

```
☑ .NET Framework 4.7.2 타겟팅 팩
☑ .NET Framework 4.8 타겟팅 팩
☑ .NET Framework 4.8 SDK
```
